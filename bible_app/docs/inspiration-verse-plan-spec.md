# ТЗ: план «Стих для вдохновения»

Документ фиксирует требования к функционалу **без привязки к конкретной реализации в коде**. Версия ТЗ: 1.0 (2026-05-19).

## 1. Цель и границы

### 1.1. Назначение

Новый тип плана чтения во вкладке **«План»**: **«Стих для вдохновения»**. Пользователь получает **локальные** push-уведомления в выбранное время с приглашением открыть стих в приложении. Дополнительно — церковные календари (православный / протестантский) и **личные особые дни** со своим набором стихов.

### 1.2. Жёсткие ограничения

| Требование | Решение |
|------------|---------|
| Серверы, FCM, облако | **Не используются** |
| Системный календарь | **Не читается** |
| Чужие данные пользователя | **Не затрагиваются** |
| Текст Библии в push | **Не дублируется** — только ссылка и контекст дня |
| По умолчанию | Функция **выключена** |
| Запрос разрешений при установке | **Нет** — только при явном включении напоминаний |

### 1.3. Вне scope первой версии (можно позже)

- Редактирование стихов церковных праздников пользователем (overrides).
- Импорт дат из Google Calendar / системного календаря.
- Повторное «напоминание», если пользователь не открыл приложение.
- Удалённые push и аналитика.

## 2. Поведение продукта

### 2.1. Выбор плана

- В списке планов (как «Параллельный», «Вера», «Любовь»…) добавляется пункт **«Стих для вдохновения»** с кратким подзаголовком, например: *«Ежедневное напоминание и особые дни»*.
- После выбора открывается **экран настроек**, а **не** сетка кварталов.

### 2.2. События на календарный день

На дату `D` формируется **список событий** для уведомлений:

1. Все праздники из **включённого** православного набора, попадающие на `D`.
2. Все праздники из **включённого** протестантского набора, попадающие на `D`.
3. Все **включённые** личные особые дни с совпадающими `месяц` + `день` (ежегодно).
4. Если список пуст — одно событие **«обычный стих дня»** (случайный стих из всей Библии).

**Правила:**

- В день с хотя бы одним праздником или личным днём событие **«обычный стих дня» не создаётся** (нет лишнего четвёртого push).
- Если включены **оба** церковных календаря и на `D` есть праздники в каждом — **отдельное уведомление на каждый** праздник.
- Если на `D` праздник **и** личный день — **два уведомления** (разные слоты, см. § 5).
- При создании личного дня на дату с уже существующим церковным праздником (включённый календарь) — **предупреждение**, сохранение **разрешено**; в этот день придут уведомления и по празднику, и по личному дню.

### 2.3. Выбор стиха

| Тип события | Источник стиха | Алгоритм |
|-------------|----------------|----------|
| Обычный день | Вся Библия (SQLite, существующий `BibleService`) | Детерминированный «случайный»: `k = hash(дата) mod N`, где `N` — число всех стихов. Один стих на весь календарный день; тот же после перезапуска приложения. **Не** заранее составленный список на год. |
| Церковный праздник | Список `verseRefs` в assets | `index = hash(дата + id праздника) mod len(verseRefs)` |
| Личный особый день | Список `verseRefs`, заданный пользователем | `index = hash(дата + id личного дня) mod len(verseRefs)` |

Минимум один стих в списке для праздника и личного дня; при добавлении личного дня без стихов — не сохранять (валидация UI).

### 2.4. Текст уведомления

Формат (книга/глава/стих — из локальной Библии, краткое имя книги как в приложении):

| Ситуация | Текст |
|----------|--------|
| Обычный день | `{ссылка} — откройте для размышления` |
| Праздник или личный день | `{ссылка} — откройте для размышления, сегодня {название}!` |

Примеры:

- `Быт 24:67 — откройте для размышления`
- `Ин 1:1 — откройте для размышления, сегодня Пасха!`
- `Пс 127:3 — откройте для размышления, сегодня День рождения жены!`

Полный текст стиха в push **не показывается**.

### 2.5. Действие по нажатию на уведомление

- Открыть приложение на вкладке **«Библия»**.
- Перейти к книге / главе / стиху из уведомления.
- Подсветить стих (существующий механизм фокуса, как из поиска / плана).

### 2.6. Блок «Сегодня» на экране настроек

- Показать ссылку на стих(и) текущего дня по тем же правилам, что и для push.
- Кнопка **«Открыть в Библии»** — без ожидания времени уведомления.
- Если сегодня несколько событий — показать список (Пасха + личный день).

## 3. Локальные уведомления и окно планирования

### 3.1. Принцип

- Используются **локальные запланированные уведомления** (`flutter_local_notifications` + `timezone` или эквивалент).
- Текст и ссылка на стих **подставляются в момент планирования**, а не вычисляются в фоне в момент срабатывания (процесс приложения может быть убит).

### 3.2. Скользящее окно 14 дней

| Событие | Действие |
|---------|----------|
| Пользователь включил напоминания | Запрос разрешений → планирование **14 дней** вперёд от «сегодня» |
| Сменил время, календари, личные дни | Отмена старых слотов категории → перепланирование 14 дней |
| Открыл приложение | Перепланирование, **не чаще 1 раза в сутки** (чтобы не нагружать слабые устройства) |
| Выключил напоминания | Отмена всех запланированных уведомлений этого плана |

### 3.3. Приложение закрыто (не в фоне)

- Уже запланированные на 14 дней уведомления **доставляет ОС** — push приходят, пока не исчерпано окно.
- Если **15+ дней** приложение не открывали — новые слоты **не добавляются**, напоминания **прекращаются** до следующего открытия и перепланирования.

### 3.4. Перезагрузка устройства

- После `BOOT_COMPLETED` (Android) желательно перепланировать при следующем запуске приложения; при реализации — зарегистрировать приёмник или полагаться на открытие приложения (минимальный вариант — только открытие).

### 3.5. Один push в день на событие

- **Без** повторного напоминания, если пользователь не прочитал / не открыл.

### 3.6. Идентификаторы уведомлений (anti-spam в шторке)

| Слот | `notificationId` (логика) | Перезатирание |
|------|---------------------------|---------------|
| Обычный стих дня | Один **постоянный** id, напр. `inspiration_daily` | Новый день **заменяет** в шторке вчерашний обычный push с тем же id |
| Церковный праздник | Уникальный на праздник, напр. `inspiration_holiday_{tradition}_{id}` | Не затирается «обычным» на следующий день (другой id) |
| Личный день | `inspiration_custom_{uuid}` | Аналогично |

**Очистка «зависших» уведомлений:** при перепланировании окна 14 дней (и при открытии приложения, раз в сутки) — **снять все активные уведомления канала «Стих для вдохновения»** (или все зарезервированные id категории), затем заново запланировать. Это предотвращает ситуацию, когда вчерашняя «Пасха» остаётся в шторке после того, как сегодня пришёл только обычный стих.

**Несколько push в один день:** разные id → в шторке одновременно видны все события дня (например, Пасха + день рождения).

**Сдвиг по времени:** при нескольких событиях в одно время допустимо срабатывание в одну секунду; при тестах при необходимости — сдвиг +60 с для второго (косметика, не обязательно в MVP).

### 3.7. Разрешения (Android / iOS)

| Разрешение | Когда запрашивать |
|------------|-------------------|
| Уведомления (`POST_NOTIFICATIONS`, Android 13+) | После включения «Ежедневное напоминание» и короткого объяснения в диалоге |
| Точное время / exact alarm (Android 12+) | Следующим шагом, если нужно стабильное время; при отказе — предупреждение о возможной задержке |
| Системный календарь | **Не запрашивать** |

**RuStore / Google Play:** локальные напоминания по явному согласию — стандартная практика. Обновить политику конфиденциальности и кратко упомянуть в описании магазина. В **Инструкции** — раздел с шагами включения и (при необходимости) исключения из оптимизации батареи.

## 4. Данные и хранение

Принципы: **быстродействие**, **работа на слабом железе**, **минимум ошибок**, согласованность с текущим проектом (`SharedPreferences`, SQLite Библии, assets).

### 4.1. Церковные праздники — assets (только чтение)

Файлы (имена условные):

- `assets/inspiration/orthodox_holidays.json`
- `assets/inspiration/protestant_holidays.json`

**Запись праздника:**

```json
{
  "id": "orthodox_christmas",
  "name": "Рождество Христово",
  "dateRule": { "type": "fixed", "month": 1, "day": 7 },
  "verseRefs": [
    { "book": "Мф", "chapter": 2, "verse": 1 }
  ]
}
```

**Подвижные праздники** — `dateRule` вида `{ "type": "easter", "offsetDays": 0 }` (смещение от даты Пасхи). Для православного и протестантского набора — **разные формулы расчёта Пасхи** (зафиксировать в реализации один раз, покрыть тестами на несколько лет).

Парсинг JSON — **лениво**: при первом открытии плана «Стих для вдохновения» или при включении напоминаний, не при старте всего приложения.

### 4.2. Настройки плана — SharedPreferences

Ключ (условный): `inspiration_plan_settings`

| Поле | Тип | По умолчанию |
|------|-----|--------------|
| `enabled` | bool | `false` |
| `notifyTimeMinutes` | int (0…1439) | `420` (07:00) |
| `useOrthodoxCalendar` | bool | `false` |
| `useProtestantCalendar` | bool | `false` |
| `lastRescheduleDate` | string ISO date | — (для «не чаще раза в сутки») |

### 4.3. Личные особые дни — SharedPreferences

Ключ (условный): `inspiration_custom_days` — JSON-массив.

| Поле | Тип |
|------|-----|
| `id` | string (uuid) |
| `name` | string |
| `month` | int 1…12 |
| `day` | int 1…31 |
| `enabled` | bool |
| `verseRefs` | массив `{ book, chapter, verse }` |

Ожидаемый объём: десятки записей — отдельная SQLite-таблица **не требуется**.

### 4.4. Отдельная таблица «стих дня» — не нужна

Достаточно детерминированного выбора + текста, записанного в запланированное уведомление.

### 4.5. Обычный день — индекс по всей Библии

- `N` = общее число стихов (из `BibleService` / метаданных БД).
- `k = hash(YYYY-MM-DD) mod N` → один запрос к БД для получения ссылки.
- **Не** держать в RAM список из 31 000+ стихов.

### 4.6. Влияние на размер APK

| Компонент | Оценка |
|-----------|--------|
| JSON праздников (оба набора) | десятки КБ |
| `flutter_local_notifications` + `timezone` | ~0,5–1,5 МБ |
| Дублирование Библии | **нет** |

## 5. Экраны и UX (эскиз)

### 5.1. Главный экран плана

```
┌─────────────────────────────────────────┐
│  ←  [Стих для вдохновения ▼]     ⋮     │
├─────────────────────────────────────────┤
│  ┌─ Ежедневное напоминание ───────────┐ │
│  │  [ ] Включить напоминания          │ │
│  │      Время:  [  07 : 00  ]        │ │
│  └────────────────────────────────────┘ │
│  ┌─ Сегодня ──────────────────────────┐ │
│  │  {ссылка(и)}                       │ │
│  │  [ Открыть в Библии ]              │ │
│  └────────────────────────────────────┘ │
│  ┌─ Церковный календарь ──────────────┐ │
│  │  [ ] Православные праздники        │ │
│  │  [ ] Протестантские праздники      │ │
│  │  [ Посмотреть список праздников ]  │ │
│  └────────────────────────────────────┘ │
│  ┌─ Мои особые дни ───────────────────┐ │
│  │  {список с ✎ 🗑}                   │ │
│  │  подпись ⚠ при совпадении с празд. │ │
│  │  [ + Добавить особый день ]        │ │
│  └────────────────────────────────────┘ │
│  ┌─ Как это работает ─────────────────┐ │
│  │  краткий текст + Инструкция       │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

Стиль: карточки и хром как у «Настройки» / тематических планов (светлая и тёмная тема).

### 5.2. Диалог «Добавить / редактировать особый день»

- Поля: название, дата (день + месяц, ежегодно).
- Список стихов + «Добавить стих из Библии» (выбор как в избранном).
- При совпадении с включённым церковным календарём — **предупреждение** (список праздников на эту дату), кнопки «Отмена» / «Сохранить».

### 5.3. Экран «Список праздников»

- Только просмотр на текущий год (фиксированные + вычисленные подвижные даты).
- Переключатель или два раздела: православные / протестантские (по включённым галочкам).

### 5.4. Первое включение напоминаний

Диалог с объяснением формата push и хранения данных на устройстве → **«Разрешить»** → системный запрос уведомлений → при необходимости шаг «Точное время».

### 5.5. Меню «⋯»

- Перейти в **Инструкцию** (раздел «Стих для вдохновения»).
- **Проверить разрешения** (если ранее отказ).

## 7. Этапы реализации

| Этап | Содержание | Критерий готовности |
|------|------------|---------------------|
| **1** | Пункт в списке планов, экран настроек (без push), блок «Сегодня», логика выбора стиха | Можно открыть стих дня из UI |
| **2** | Assets: православный и протестантский JSON, расчёт Пасхи, экран списка праздников | Даты на год совпадают с эталоном для тестовых лет |
| **3** | Личные дни: CRUD, предупреждение о совпадении | Два push в один день в тестовом сценарии |
| **4** | Локальные уведомления, разрешения, окно 14 дней, id слотов, очистка шторки | 14 дней без открытия app → push идут; 15-й — нет |
| **5** | Инструкция, privacy policy, описание для RuStore | Тексты согласованы с поведением |

## 8. Тестирование (чеклист)

- [ ] Обычный день: текст без «сегодня …!»; стих стабилен в течение дня.
- [ ] Православная Пасха: текст с названием праздника; стих из списка праздника.
- [ ] Оба календаря включены, на одну дату два разных праздника — два push.
- [ ] Личный день + праздник — два push, два id в шторке.
- [ ] Создание личного дня на дату праздника — предупреждение, сохранение, два push в этот день.
- [ ] Включение напоминаний — запрос **только** после toggle, не при первом запуске app.
- [ ] Приложение убито 3 дня — push приходят; 15 дней без открытия — прекращаются.
- [ ] Открытие app на 10-й день — снова планируются 14 дней вперёд.
- [ ] Tap по push → правильная книга/глава/стих с подсветкой.
- [ ] Минимальный и максимальный размер шрифта — push не зависят (только ссылка).
- [ ] Перезагрузка телефона (Android) — после открытия app расписание восстановлено.

## 9. Открытые вопросы на реализацию

1. **Точные формулы** православной и протестантской Пасхи — зафиксировать в коде с комментарием и unit-тестами (3–5 эталонных лет).
2. **Состав JSON** — полный список праздников MVP (согласовать с заказчиком: только великие или + двунадесятые и т.д.).
3. **Канал уведомлений Android** — имя и важность (не будить ночью: время задаёт пользователь).
4. **Лимит iOS** (~64 запланированных) — при 14 днях × несколько событий в день держать суммарно < 64 или планировать короче на iOS.
